# CSS 模块化架构说明

## 文件结构

```
css/
├── main.css              # 主入口文件，导入所有模块
├── base.css              # 基础样式和CSS变量
├── sidebar.css           # 侧边栏相关样式
├── main-content.css      # 主内容区域和欢迎页面
├── chat-input.css        # 聊天输入框相关样式
├── upload.css            # 上传页面样式
├── responsive.css        # 响应式设计
├── utilities.css         # 工具类和动画
└── style.css             # 原始文件（可以删除）
```

## 模块说明

### 1. base.css

- CSS 变量定义（颜色、尺寸、过渡效果等）
- 全局样式重置
- 基础滚动条样式
- 主容器布局

### 2. sidebar.css

- 侧边栏主体样式
- Logo 区域
- 功能按钮
- 历史对话内容
- 存证记录区域
- 用户信息区域

### 3. main-content.css

- 主内容容器
- 欢迎页面样式
- 页面标题和描述

### 4. chat-input.css

- 聊天输入框容器
- 文本域样式
- 滚动条定制
- 输入框按钮（加号、发送）
- 白色挡板样式

### 5. upload.css

- 上传页面布局
- 文件拖拽区域
- 文件预览组件
- 上传按钮样式
- 文件列表样式

### 6. responsive.css

- 平板设备适配（768px 以下）
- 手机设备适配（480px 以下）
- 响应式布局调整

### 7. utilities.css

- 工具类（.hidden, .fade-in 等）
- 动画定义
- 加载状态样式

## 使用方式

HTML 中只需要引入 `main.css`：

```html
<link rel="stylesheet" href="css/main.css" />
```

`main.css` 会自动导入所有必要的模块文件。

## 优势

1. **模块化**：每个功能模块独立，便于维护
2. **可重用性**：组件样式可以在其他项目中复用
3. **易于调试**：问题定位更精确
4. **团队协作**：不同开发者可以同时编辑不同模块
5. **按需加载**：未来可以根据页面需求选择性加载模块

## 注意事项

- 确保所有 CSS 变量在 `base.css` 中定义
- 模块间的依赖关系要明确
- 避免模块间的样式冲突
- 保持模块的职责单一
